# Проект по работе с XML и XSLT

Данный проект демонстрирует работу с XML-данными, выполнение XSLT-преобразований и взаимодействие с базой данных MS SQL через C#.

## Инструкция по настройке и запуску

1.  **XSLT-шаблон**: В папке с исполняемым файлом (`.exe`) должен находиться файл `Employees.xslt`. Если он отсутствует, скопируйте его из папки `XML` вашего решения.
2.  **База данных**:
    *   Предварительно обновите строку подключения `sqlConnection` в файле `DataBase.cs`.
    *   Для упрощения развертывания база данных MS SQL реализована в минималистичном виде.
    *   Если БД отсутствует, нажмите кнопку **«Создать базу данных»** в интерфейсе программы для её автоматической генерации.
3.  **Использование**: 
    *   В главном окне последовательно нажимайте кнопки сверху вниз.
    *   Для восстановления эталонного файла `Data1.xml` используйте соответствующую квадратную кнопку.

---

## Основной функционал

### I. XSLT-преобразование
Создан шаблон, который на основе входных файлов `Data1.xml` или `Data2.xml` формирует результирующий файл `Employees.xml`.

### II. Приложение на C#
Программа реализует следующие задачи:
1.  **Трансформация**: Выполняет XSLT-преобразование из пункта I.
2.  **Обогащение данных (XSLT)**: После преобразования добавляет в элемент `Employee` атрибут, содержащий сумму всех значений `amount/@salary` для данного сотрудника.
3.  **Обновление исходных данных**: В файл `Data1.xml` (элемент `Pay`) дописывается атрибут, отражающий общую сумму всех выплат `amount`.
4.  **Графический интерфейс (GUI)**:
    *   Кнопка запуска процесса обработки.
    *   Отображение списка всех `Employee`.
    *   Вывод суммы всех выплат (`amount`), сгруппированных по месяцам (`month`).
5.  **Редактирование**: Возможность добавления новых данных для строки `item` в файл `Data1.xml` с последующим автоматическим пересчетом всех показателей (пункты 1–4).

## Описание логики XSLT

*   **`<xsl:stylesheet>`**: Определение таблицы стилей XSLT (версия 1.0).
*   **`<xsl:output method="xml" indent="yes"/>`**: Установка выходного формата XML с отступами для удобства чтения.
*   **`<xsl:template match="/">`**: Шаблон, применяемый к корневому элементу входного документа.
*   **`<xsl:for-each select="Data/Employee">`**: Итерация по всем сотрудникам в узле `Data`.
*   **`<Employee>`**: Генерация нового узла в выходном файле.
*   **`<xsl:value-of>`**: Извлечение значений (например, `FullName` или `Position`) из исходного файла и их запись в соответствующие поля выходного XML.
